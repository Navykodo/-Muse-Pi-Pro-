# 联网搜索与网页读取技能

## 触发场景

用户出现以下意图时使用本 skill：

- “联网搜索 / 网上查 / 搜索一下 / 帮我查资料”
- “最新 / 今天 / 最近 / 最近一周 / 实时 / 当前”
- “查新闻 / 央视网 / 新华社 / 官网 / 网页链接”
- 天气、气温、降雨、台风、空气质量、穿衣建议等需要实时信息的问题

普通常识、闲聊、纯本地硬件控制不需要联网搜索。

## 工具选择

优先使用当前可用的联网搜索和网页读取工具：

1. 有 `web_search_tool` 时，先用它搜索。
2. 需要核验正文、时间或来源时，再用 `browser` / 网页读取能力打开结果页。
3. 如果搜索工具不可用或联网失败，直接说明联网失败原因，不要编造结果。

不要为了联网搜索调用本地 Hardware API，也不要用 `http_request` 请求普通网页，除非用户明确给的是本地 API。

## 搜索策略

- 新闻类：关键词加时间范围或来源限制，例如“人工智能 最近一周 央视网”“机器人 新华社 今天”。
- 指定网站：用户说央视网、新华社、官网时，优先搜索或打开该来源，不要换成泛搜索结果冒充。
- 天气类：优先查询用户所在城市或用户明确给出的地点；地点不明确时，可按用户常用所在地或直接简短询问地点。

## 输出要求

每条网页/新闻结果尽量包含：

- 标题
- 来源
- 发布时间或更新时间
- 链接
- 一句话摘要

最终回复要像语言助手一样自然简短。详细链接很多时保留关键链接即可。

## 天气与智能家居联动

天气相关请求可以联动 `smart-home-tools`，但要分清建议和控制：

- 用户只问天气：只回答天气，可顺带一句生活建议。
- 用户问“根据天气要不要开空调/开灯”：先查天气，再读取 `smart-home-tools`，查询空调/灯状态，给建议。
- 用户明确说“根据天气帮我调空调 / 太热就开空调 / 天黑就开灯”：查天气和设备状态后，按 `smart-home-tools` 控制。
- 用户没有明确授权控制设备时，不要直接开关空调或灯；用一句话询问，例如：“外面比较热，现在空调是关的，要不要我帮你开到 26 度制冷？”

联动建议示例：

- 高温：建议空调制冷 26 度、自动风。
- 低温：建议空调制热 26 度或保持关闭。
- 潮湿/雨天：建议除湿模式。
- 天色暗、暴雨或用户说房间暗：可建议或按用户授权打开灯。
